◆ FileHeader

enum ftFlags::FileHeader

FileHeader defines conditional flags that are set based on the file's header string.

Enumerator
FH_ENDIAN_SWAP 

Is a flag that indicates that the byte order in the file is different than the byte order of the platform that is loading the file.

FH_CHUNK_64 

Is a flag that indicates that the chunk size in the file was saved on a 64 bit platform.

FH_VAR_BITS 

Is a flag that indicates that the chunk size in the file does not match the chunk size on the current platform.

Enumerator
FS_STATUS_MIN 

Start code for enumerated values.

FS_INV_INSERT 

Invalid insert. This is set when inserting a chunk into the pointer lookup table.

FS_BAD_ALLOC 

Invalid allocation. Used to catch null values that are returned by malloc.

FS_INV_VALUE 

Invalid Value.

FS_INV_READ 

Invalid read. A stream read failed.

FS_INV_LENGTH 

Invalid size. An expected size constraint was not met.

FS_INV_HEADER_STR 

Invalid header. An unexpected header was read.

FS_TABLE_INIT_FAILED 

Table read error. Failed to read any structures from the table.

FS_OVERFLOW 

A diagnostics check error. A zeroed block of memory contains something other than zeros.

FS_FAILED 

A general fail status.

FS_CODE_ERROR 

If this status is returned, it means that every byte in the code did not match the following criteria:

upper case characters
lower case characters
ASCII digits [0,9]
underscore character '_'
space character ' '

Assuming that the code was constructed properly, then this most likely means that some data is out of alignment with what was computed and or reported.

enum ftFlags::WriteMode

WriteMode is a flag that is used to control how ftCompiler formats the output in write methods.

Enumerator
WRITE_ARRAY 

Writes the table as a c/c++ array.

The table is written back to c/c++ to be compiled as a source file.

const unsigned char ${TableName}Table[] = {
...
}
const int ${TableName}Len=sizeof(${TableName}Table);
WRITE_STREAM 

Writes the table as binary memory.
